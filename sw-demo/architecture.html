GitHub Pages Backend Emulation

The hosted page stays static, but the service worker takes ownership of demo URLs under /sw-demo/ and returns the same route shapes that the Node server returns locally.

What GitHub Pages does
  • Hosts static files from the repo.
  • Serves the worker module and generated demo modules.
  • Does not run Node, hold memory, or stream from a server.
What the worker mimics
  • Query-param driven rendering.
  • /_async/partial/ProductCard JSON.
  • /_async/partial/edge-segment JSON.
  • Demo cache counters, delays, and stream-like chunking.
What remains different
  • Cache state is per browser, not shared server state.
  • The real proof path is still server.mjs.
  • The reset URL clears stale worker state for debugging.
